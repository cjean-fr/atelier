# @cjean-fr/jsx-string-postcss

A plugin for the `@cjean-fr/jsx-string` ecosystem that allows you to transform rendered HTML by injecting CSS generated by PostCSS (like Tailwind CSS v4).

Built for modern web standards: Node.js, Edge, Deno, and Bun.

## Why Post-processing?

The `@cjean-fr/jsx-string` engine remains lean, pure, and dependency-free, focusing exclusively on rendering JSX to string safely and fast.
Styling tools, such as Tailwind CSS v4, often require PostCSS to parse source files and extract used utility classes. By using a "Post-processing" approach, we keep the core rendering fast while injecting generated styles exactly where they belong in your HTML.

## Installation

```bash
bun add @cjean-fr/jsx-string @cjean-fr/jsx-string-postcss postcss
```

_(You may also use npm, yarn, or pnpm)_

## Usage

Here is an example using Tailwind CSS v4:

```javascript
import { renderToString } from "@cjean-fr/jsx-string";
import { withPostCss } from "@cjean-fr/jsx-string-postcss";
import tailwindcss from "@tailwindcss/postcss";

// For Tailwind v4

// 1. Your typical component
const App = () => (
  <html lang="en">
    <head>
      <title>My Styled App</title>
    </head>
    <body className="bg-slate-900 p-4 text-white">
      <h1 className="text-3xl font-bold">Hello World</h1>
    </body>
  </html>
);

// 2. Render and inject styles
const html = await renderToString(<App />);

// 3. Post-process with Tailwind
const styledHtml = await withPostCss(html, {
  css: '@import "tailwindcss";', // For Tailwind v4
  plugins: [tailwindcss()],
});

console.log(styledHtml);
```

### Options

`withPostCss(html, options)` accepts the following options:

- `plugins` (Array): An array of PostCSS plugins to apply. Default is `[]`.
- `css` (String): A base CSS string to parse (e.g., `@import "tailwindcss";` or `@tailwind utilities;`). Default is `''`.
- `useCache` (Boolean): Whether to cache the processed output to speed up subsequent renders. The cache key is generated securely using the Web Crypto API. Default is `true`.

## Features

- **Web Crypto API Cache**: Uses `globalThis.crypto.subtle` for extremely fast, non-blocking hashing to cache results.
- **Auto-Injection**: Automatically injects the generated `<style>` tag into the `<head>` of the HTML document. If no `<head>` is found, it prepends the style tag.
- **Edge Compatible**: No Node.js specific APIs. It works seamlessly on Edge runtimes.
