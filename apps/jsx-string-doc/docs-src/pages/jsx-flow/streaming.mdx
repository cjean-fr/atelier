# Streaming

Send the HTML shell immediately and stream deferred fragments inline in the
same HTTP response.

## renderStream

Returns a `ReadableStream<string>`. The shell chunk is emitted first; fragment
chunks follow as they resolve. The adapter is optional — it defaults to
`NativeAdapter` (inclut un polyfill ~550B).

<CodeExample src="jsx-flow/deferred-stream.tsx" />

## Node HTTP server

Pipe the stream into any Node.js `http.ServerResponse` with `for await`. Each
chunk is written as it becomes ready — the shell flushes immediately, deferred
fragments arrive later.

<CodeExample src="jsx-flow/node-server.tsx" />

The example uses `http.createServer`, but the same pattern works with **Express**,
**Fastify**, or any framework built on `node:http`. The key is consuming the
`ReadableStream` chunk by chunk and writing each one to the response without
buffering.

## Components

jsx-flow provides four primitives for deferred content. See the
[Components](/jsx-flow/components) page for the full API:

- **`<Slot name="…">`** — a named hole with fallback content in the shell.
- **`<Fill target="…">`** — pushes deferred content into a slot.
- **`<Defer name="…">`** — deferred content with a built-in placeholder.
- **`<ClientFetch src="…">`** — client-side fetch placeholder.

## `<ClientFetch>` — client-side fetch

The browser fetches the fragment from a URL after the shell lands. No
server-push required — works with any static file server.

<CodeExample src="jsx-flow/deferred-src.tsx" />

## Async iterables — streaming sequences

An `AsyncIterable` returned by the factory streams one patch per item:

```tsx
<Fill target="feed" merge="append">
  {() => liveRows()}
</Fill>
```

Each `yield` produces one patch with the configured `merge` type. Supports:
`"replace"` (default), `"append"`, `"prepend"`, `"before"`, `"after"`.
