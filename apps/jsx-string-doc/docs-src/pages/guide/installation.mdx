# Installation

jsx-string has zero runtime dependencies. Install it and configure TypeScript —
that's all.

## Package manager

<Tabs syncKey="pkg-manager" tabs={[

  { label: 'bun', content: <CodeBlock code="bun add @cjean-fr/jsx-string" language="bash" /> },
]} />

## Optional: @types/react

Installing `@types/react` gives you per-element attribute autocomplete (e.g.
`src` on `<img>`, `href` on `<a>`). It is not required — jsx-string works
without it.

## Deno

jsx-string works with Deno's built-in JSX transform. Configure the import
source in `deno.json` so you don't need explicit imports in every file:

<CodeExample src="guide/installation/deno.json" />

The same configuration applies to all `*.tsx` and `*.jsx` files in your
project. No additional plugins or bundler configuration needed.

## Vite plugin (precompile)

The Vite plugin transforms JSX at build time into tagged template literals,
eliminating runtime parsing overhead. The output is byte-identical to the
dynamic path — you can toggle it on and off without changing anything.

<CodeBlock
  code={`import precompile from "@cjean-fr/vite-plugin-precompile";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [precompile()],
});`}
  language="ts"
/>

Use it for static sites, docs, or anything where the JSX tree is known at build
time. Works alongside the dynamic path — you can precompile some files and
runtime-render others.

## ESLint plugin

The ESLint plugin catches common mistakes — unsafe `raw()` calls, event
handlers that will be silently dropped, and attributes incompatible with static
rendering.
